What is cssnext ?

This is a CSS transpiler that allows you to use tomorrow's CSS syntax today. It transforms CSS specs that are not yet implemented in popular browsers into more compatible CSS. You should understand that you can code directly in CSS4+ and it will be transformed to CSS3. This is not a classic CSS preprocessor, but can replace one. You can use variables (by using custom property syntax, limited to top-level :root) with math. You can create custom media queries for a more maintainable responsive CSS that works on the devices you specify. You can do some crazy color manipulations with all the new color functions and available modifiers. You can also forget about vendor prefixes. There is also some others stuff included and others that will be, when the specifications evolve. You can basically write future-proof code today and forget old preprocessor specific syntax.

Why cssnext?

Future-proof code

In a near future, browsers will implement new CSS specifications. As time goes, cssnext will remove some transformations that won't be necessary anymore. And maybe one day, you will be able to completely remove cssnext from your workflow without touching your actual CSS.

It's fast. Lightning fast.

cssnext uses postcss 3 which has the fastest css parser. It's faster than rework and Myth. And Sass and other similar tools.

Active development

No long wait for releases. A bug fixed will be released ASAP. Minor features will never wait too much.
See that by yourself on cssnext & postcss. activities graphs.
cssnext use lots of postcss plugins, so check them out too.

Based on NPM ecosystem

cssnext uses npm packages that should respect semver.
Bugfixes are pushed often. Minor changes don't break your code. Major releases are, well, major and are not supposed to happen a lot.

Follow @cssnext to get latest news or join #cssnext on irc.freenode.net if you have any questions.
